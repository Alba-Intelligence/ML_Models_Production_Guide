# Revision: 2026-05-09 example matrix

## Trigger

After the contracts, domains, topologies, and default stack decisions were drafted, the next high-confidence step was to define an architecture-aligned example inventory.

## Artifact added

- `docs/wiki/architecture/example-matrix.md`

## What it establishes

- a first-pass inventory of 10 examples
- mapping from examples to lifecycle stages
- mapping from examples to domains and topologies
- implementation priority order
- an explicit rule against random tool demos

## Architectural effect

The example matrix makes it easier to:

- sequence implementation later
- keep examples aligned with contracts and topologies
- avoid premature or disconnected code artifacts
