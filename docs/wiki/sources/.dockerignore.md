# Source summary: .dockerignore

## Role

Constrains Docker build context to avoid copying local virtual environments, docs output, caches, and run artifacts.

## Practical implication

Builds are leaner and avoid accidental inclusion of local-only state (`.venv`, `_docs`, `mlruns`, and git metadata).
