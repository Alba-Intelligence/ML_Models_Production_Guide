# Source summary: .github/workflows/ci.yml

## Role

Defines repository CI for pull requests and main-branch pushes.

## Pipeline behavior

1. installs Python 3.13, uv, and Quarto
2. runs `uv sync`
3. generates Docker artifacts from Nix Terranix definitions when `nix` is available
4. exports nbdev modules via `uv run nbdev-export --path nbs/`
5. renders docs via `quarto render . --no-execute`
6. runs unit tests via `uv run python -m unittest discover -s tests -q`

## Practical implication

Docs freshness checks are CI-enforced because rendered `_docs/nbs/*.html` is produced before tests execute.
